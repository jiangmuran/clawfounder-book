# WORKFLOWS

## 完整流程概览

ClawFounder 面向的是项目推进，而不是单点执行。

完整流程可抽象为：

`Idea -> Spec -> Prototype -> Repo -> Launch -> Growth -> Feedback`

---

## 1. Idea Intake

输入：

- 新点子
- 问题定义
- 目标用户
- 初步价值判断

目标：

- 确认是否值得做
- 明确核心问题和最小可行方向

输出：

- 方向判断
- 产品定位
- 目标用户画像

---

## 2. Spec Generation

输入：

- 已确认方向的项目想法

目标：

- 形成结构化规格，而不是直接进入开发

输出：

- 白皮书
- MVP 范围
- 用户故事
- 执行清单

---

## 3. Prototype & Repo

输入：

- 产品规格
- 页面结构
- 功能清单

目标：

- 调用编码代理生成首版原型
- 初始化 GitHub 仓库

输出：

- 原型代码
- GitHub 仓库
- 可迭代的项目资产

---

## 4. Launch Assets

输入：

- 原型代码
- 项目定位
- 品牌表达

目标：

- 把项目从“能运行”升级为“能展示”

输出：

- 官网
- 项目说明页
- 访问链接
- 海报 / 主视觉

---

## 5. Growth Loop

输入：

- 项目现状
- 外部信息
- 会议纪要 / 论坛内容

目标：

- 把外界输入转化为新的传播和增长动作

输出：

- 文章草稿
- 推广计划
- 周报 / 复盘
- 下一个项目机会

---

## 6. Device Bridge

输入：

- 需要移动端参与的任务
- 设备级动作需求

目标：

- 通过 ADB 接管独立手机
- 把移动端动作纳入统一工作流

输出：

- 设备执行动作完成
- 工作流断点减少
- 现实接口层补齐

---

## 7. Feedback 回流

每一轮执行结束后，项目不会停在交付状态，而是进入反馈回流：

- 哪些动作有效
- 哪些产物最有价值
- 哪些环节摩擦最大
- 下一轮是否值得继续投入

这使得系统从“自动化工具”转向“项目推进系统”。
