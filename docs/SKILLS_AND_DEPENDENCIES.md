# SKILLS AND DEPENDENCIES

本文件说明 ClawFounder 当前依赖的关键能力组件，以及它们在整套流程中的职责。

---

## 1. OpenCode / Coding Agent

### 作用

- 根据产品规格快速生成原型
- 补全静态站点、页面结构和仓库内容
- 适合把 idea 快速推进到可运行版本

### 在流程中的位置

- `Spec Generation` 之后
- `Prototype & Repo` 阶段的核心执行器

### 典型输入

- README / 白皮书 / 页面需求
- MVP 范围
- 视觉和文案约束

### 典型输出

- HTML / CSS / JS 原型
- README 与 docs 补全
- 可部署的静态项目

---

## 2. dvcode / Nano Banana

### 作用

- 生成主视觉
- 生成海报背景与产品插图
- 提升比赛项目的视觉完成度

### 在流程中的位置

- `Launch Assets` 阶段
- 海报、说明页、宣传资源阶段

### 当前注意事项

- 易受官方配额影响
- 应在成功后尽快把图片落盘到仓库
- 更适合生成主视觉，不适合单独承担整页信息排版

---

## 3. GitHub Pages

### 作用

- 承载项目说明页
- 提供评委可直接访问的公开链接

### 在流程中的位置

- `Launch Assets` 阶段

### 典型输出

- `index.html` 可访问页面
- 海报与图片资源可直接访问的路径

---

## 4. ADB Device Bridge

### 作用

- 接管一台独立手机
- 把移动端专属动作纳入工作流
- 补足桌面流程中的现实接口断点

### 在流程中的位置

- `Device Bridge` / `Real-world Interface Layer`

### 关键价值

- 不是“多控制一台设备”
- 而是让系统在需要移动端参与时仍然保持闭环

---

## 5. 内容飞轮能力

### 作用

- 把会议纪要、论坛信息、日常观察转为可传播内容
- 支撑项目从上线到增长的后续阶段

### 在流程中的位置

- `Growth Loop`

### 典型输出

- 文章草稿
- 推广文案
- 周报 / 复盘

---

## 6. 当前建议的最小演示配置

如果只想录一个完整演示视频，最低建议准备：

1. 一个可展示的 idea
2. 一套规格文档或 README
3. 一个原型页面
4. 一个 GitHub Pages 链接
5. 一张主视觉或海报
6. 一段 ADB 独立手机桥接的说明或片段

这样就足以让评委理解：

- 它不是聊天助手
- 它是项目推进系统
