# DEMO SCRIPT

以下是一份适合 2.5 ~ 3 分钟录屏演示的视频脚本。

---

## 0:00 - 0:20

### 开场问题

可以直接说：

> 对于独立开发者或 One-Person Company 来说，真正难的不是某个任务做不出来，而是项目总在中途失速。
> ClawFounder 想解决的是：如何把一个人的创业流程，稳定推进到上线、传播与复盘。

画面建议：

- 先展示海报
- 再切到项目说明页首页

---

## 0:20 - 0:45

### 展示项目说明页

重点讲：

- 这是一个面向 OPC 的 AI 创业流水线
- 它不是聊天助手
- 它是项目推进系统

画面建议：

- 首页 Hero 区
- 让标题和主视觉先出现

---

## 0:45 - 1:20

### 展示完整流程

讲清楚 5 个阶段：

1. Idea Intake
2. Spec Generation
3. Prototype & Repo
4. Launch Assets
5. Growth Loop

画面建议：

- 滚到时间线
- 滚到流程图 / 交互演示

旁白重点：

> ClawFounder 不是把功能堆在一起，而是让每一步都有明确接棒关系。

---

## 1:20 - 1:50

### 展示仓库与工程感

重点讲：

- 仓库里不仅有页面，还有架构文档、流程文档、脚本模板
- 说明这个项目不是单页概念图，而是有工程组织能力的系统

画面建议：

- 打开 GitHub 仓库
- 展示 `docs/` 和 `scripts/`

---

## 1:50 - 2:20

### 强调 ADB 独立手机桥接

旁白建议：

> 普通 Agent 演示往往停留在浏览器和代码里。
> ClawFounder 进一步通过 ADB 接管独立手机，把移动端专属动作纳入统一工作流，这就是它的现实接口层。

画面建议：

- 页面中的架构图
- `docs/ADB_DEVICE_BRIDGE.md`

---

## 2:20 - 2:50

### 展示视觉与海报

旁白建议：

> 在对外展示层，ClawFounder 还会结合 dvcode / Nano Banana 生成主视觉、海报和项目视觉资产，让项目具备真正可评审、可传播的形态。

画面建议：

- 展示 `poster-v2.png`
- 展示 `assets/nano/nano-opc.png`

---

## 2:50 - 3:00

### 收束

一句话结尾：

> ClawFounder 不是一个更强的助手，而是一套把一个人的创业流程编译成 AI 流水线的系统。

---

## 录制建议

### 推荐录屏顺序

1. 海报
2. 项目页首页
3. 时间线
4. 架构图
5. GitHub 仓库
6. ADB 文档
7. 海报收尾

### 推荐风格

- 镜头稳定
- 滚动慢一点
- 不要一次展示太多窗口
- 每 20~30 秒只讲一个重点
