# TUTORIAL

本教程面向两类读者：

1. 想快速理解 ClawFounder 工作方式的人
2. 想录制演示视频或复现实验流程的人

---

## 一、从 0 到 1 的最简使用方式

### Step 1：确定一个项目 idea

先准备一个简短的问题陈述：

- 想解决什么问题
- 面向谁
- 为什么值得做

### Step 2：整理成规格

将 idea 进一步写成：

- 白皮书摘要
- MVP 范围
- 页面结构
- 成功标准

### Step 3：调用编码代理生成原型

参考脚本：

```bash
bash scripts/run_opencode_prototype.sh /path/to/project
```

### Step 4：部署说明页

```bash
bash scripts/deploy_pages.sh
```

### Step 5：补视觉资源

如果需要主视觉或海报背景：

```bash
bash scripts/run_nanobanana_poster.sh
```

### Step 6：补充现实接口层说明

如果项目涉及移动端桥接，可使用：

```bash
bash scripts/capture_device_bridge_notes.sh
```

---

## 二、推荐演示顺序

如果用于比赛展示，建议按以下顺序演示：

1. 先展示项目问题
2. 再展示说明页
3. 再解释完整流程
4. 再强调 ADB 独立手机桥接
5. 最后展示海报和仓库文档

这样更容易让评委先理解“为什么需要它”，再理解“它怎么做”。

---

## 三、常见误区

### 误区 1：一开始就展示很多功能

这样容易让评委觉得只是工具堆叠。

### 误区 2：只展示代码，不展示项目推进链

比赛更关心系统的完整度，而不仅是代码生成能力。

### 误区 3：把移动端桥接讲成具体平台技巧

更推荐讲成：

- 现实接口层
- 独立手机接管
- 移动端动作桥接

这样叙事更稳，也更专业。
